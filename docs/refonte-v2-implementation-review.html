Guide de revue · refonte v2

Ce que ce PR change.

Ce PR n'est pas une retouche : c'est la coupe franche décrite dans docs/refonte-v2.html. L'ancien dicton hexagonal multi-providers devient un petit runtime Groq-only : hotkey, audio chunké pendant la parole, transcription Groq, un seul cleanup final, puis paste dans l'application active.

Runtime
14 fichiers top-level dans src/dicton/, 713 LOC mesurées.
Provider
Groq seul : Whisper STT + cleanup LLM via le même client HTTP/2.
Validation
./scripts/check.sh all passe : lint, tests, build.
Contrat reviewer
Vérifier la simplicité, le flux async, les frontières OS et la suppression de surface.
01 · intention

Pourquoi autant de suppressions ?

L'ancien code absorbait l'incertitude : ports, adapters, factories, DI, fournisseurs STT/LLM multiples, notifications, web config UI, packaging .deb/PyInstaller. La spec verrouille l'inverse : un provider, une clé, un endpoint, une UX terminal. Le plus gros risque n'était donc pas d'ajouter une fonctionnalité, mais de garder l'architecture faite pour un futur qui n'est plus voulu.

Supprimé
hexagonal core, adapters multi-provider, notifications, web UI, visualizers GTK/Vispy, tests liés à ces surfaces.
Gardé
le mécanisme important : chunking sur silences pendant l'enregistrement, avec overlap et max cut.
Ajouté
wizard Rich, config TOML, stats JSONL, direct REST Groq en HTTP/2, pygame visualizer minimal.
02 · pipeline

Le chemin critique tient dans pipeline.py.

HOTKEY DOWN
  ├─ GroqClient(api.groq.com, http2=True) + prewarm /models
  ├─ Chunker.reset()
  ├─ Visualizer.start()
  └─ sounddevice RawInputStream feeds PCM16 frames

WHILE SPEAKING
  ├─ Chunker.feed(frame)
  ├─ if silence boundary or max duration:
  │    └─ create_task(transcribe(client, wav_chunk))
  └─ visualizer receives same frame

HOTKEY UP
  ├─ Chunker.flush() for tail
  ├─ await all STT tasks
  ├─ joined = " ".join(transcripts)
  ├─ cleanup_text(client, joined, configured_model)  # one call only
  ├─ paste(cleaned)
  └─ append JSONL stat

Review invariant: the final cleanup must happen once, after all chunk transcripts are joined. Per-chunk cleanup would break the refonte latency/quality premise.

05 · ce qui a bougé dans le packaging

Les dépendances racontent la même histoire.

pyproject.toml retire les SDKs/provider stacks et optional extras legacy. Le runtime annoncé est maintenant explicite : sounddevice, pynput, pygame, httpx[http2], platformdirs, typer, rich, et evdev seulement sur Linux. Le versioning passe par hatch-vcs pour coller au modèle tag git → PyPI.
